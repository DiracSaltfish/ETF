# 159605 盘中估值与赎回套利方案

更新日期：2026-07-11。本文定义 `159605`（广发中证海外中国互联网30 QDII-ETF）的估值、结算复核及对冲框架，供后续把该基金接入网站时复用。

> 结论先行：159605 **目前不是实物篮子赎回**。它采用全现金替代，赎回人不会取得 30 只股票。因此不能照搬 159518“买 ETF、空单对应股票、赎回后以收到的股票回补”的无风险锁定逻辑。盘中看到的折价只是一笔有跨市场、执行和汇率剩余风险的赎回交易；在取得自己的实际现金替代退款数据并完成拟合前，不应称为确定性套利。

## 2. 结算机制：真正需要估的是什么

对 T 日确认成功的赎回，基金管理人对每只“允许现金替代”的美国、香港证券：

1. 以 T 日实际卖出成交价扣除相关费用后的人民币金额计算；
2. 当日未卖出的证券，以 T 日收盘价（当日无交易时用最近交易日收盘价）计算；
3. 汇总得到赎回现金替代金额；
4. 再根据 T 日实际现金差额清算。

招募说明书约定赎回替代金额在 `T+1` 日后的第 7 个“深交所与主要境外市场共同交易日”内交收；T 日现金差额在 `T+2` PCF 公布，赎回时正数由基金支付给投资者、负数由投资者支付给基金。[招募说明书，第 67–68 页](http://pdf.dfcfw.com/pdf/H2_AN202607051826729057_1.pdf)

定义：

\[
S_T=\sum_{i\in HK,US} q_i\times \text{基金实际卖出/收盘结算价}_{i,T}\times FX^{settle}_{i,T}
\]

\[
C_T=\text{实际现金差额}_{T}
\]

\[
R_T=S_T+C_T+\text{其他对价}-\text{基金/券商向赎回人收取的费用}
\]

其中 `R_T` 才是一个最小申赎单位最终的赎回收入。对 159605：

- `S_T` 是基金经理代理卖出（或收盘价兜底）的现金替代款，**不是交付给你的股票**；
- `C_T` 是真实现金腿，不能因为金额小而在收益归因中忽略；
- T 日 PCF 的 `EstimateCashComponent` 只是事前冻结资金和盘中估值的估计现金腿，不能替代后续的 `C_T`；
- 当前 PCF 同行的 `CashComponent` 属于该文件的 `PreTradingDay`，不能误当作当前 T 日实际值。

## 3. 正确的盘中估值口径

### 3.1 两个必须并存的数

**A. 盘中可交易估值（用于决定是否买入 ETF）**

\[
\widehat V_t=\sum_{i\in HK}q_iP^{HK}_{i,t}\widehat{FX}^{HKD/CNY}_t+
\sum_{i\in US}q_iP^{US}_{i,t}\widehat{FX}^{USD/CNY}_t+\widehat C_T-\widehat{Cost}_T
\]

- `q_i`：当日 159605 PCF 的 `ComponentShare`；只取 30 个成分股，排除 `159900 申赎现金` 虚拟行。
- `P`：港股、美股的可执行价格。保守收入估值用基金卖出可得到的 Bid/预估滑点，研究展示可另列 Mid。
- `FX`：赎回 T 日的人民币折算固定使用外汇交易中心公布的 `USD/CNY`、`HKD/CNY` **收盘参考价**。港股直接乘 `HKD/CNY`、ADR 直接乘 `USD/CNY`；不得用 IB 的 `USD/CNH ÷ USD/HKD` 离岸交叉盘替代。
- `\widehat C_T`：当日 PCF `EstimateCashComponent`，同时展示其历史误差缓冲。
- `\widehat{Cost}_T`：预计卖出冲击、交易费、汇兑/结算费、赎回代理佣金及借券费；没有实盘样本时必须单列为不确定项，不能填零后称为套利收益。

每份估计净值：

\[
\widehat{NAV}_t=\widehat V_t/1{,}000{,}000
\]

买入并赎回方向的保守折溢价率：

\[
\text{可实现折溢价率}_{t}=\frac{\text{159605 买入可成交 Ask}}{\widehat{NAV}^{low}_t}-1
\]

其中 `NAV^low` 使用成分股 Bid、保守汇率、现金差额误差缓冲和全部费用。只有该值为足够大的负数，才有研究价值。不能用深交所冻结的 IOPV 代替：招募说明书说明该 IOPV 在日内为固定值，且只是 T-1 净值参考。[招募说明书，第 14、59 页](http://pdf.dfcfw.com/pdf/H2_AN202607051826729057_1.pdf)

**B. 最终结算估值（用于事后复核）**

\[
V^{settle}_T=\text{实际现金替代退款}+C_T
\]

以券商流水的实际退款和后续 PCF 的 `CashComponent` 为准。若券商流水的“退款”已包含现金差额，必须先与券商确认字段含义，避免重复相加。

### 3.2 汇率口径：使用赎回 T 日外汇交易中心收盘价

159518 的人民币换汇经验不能直接移植到 159605。对本方案，所有事后估值、开平仓回测及 KWEB 名义金额均固定使用赎回 T 日外汇交易中心的两条官方收盘参考价：

| 腿 | 固定换算 | 禁止使用 |
| --- | --- | --- |
| 美国 ADR、KWEB 美元名义 | `USD/CNY` 的 CFETS `CLOSE` | IB `USD/CNH`、盘中离岸美元汇率 |
| 港股名义 | `HKD/CNY` 的 CFETS `CLOSE` | 以 `USD/CNH ÷ USD/HKD` 推导的离岸交叉盘 |

研究数据目录必须保存两条原始 CFETS `CLOSE` 记录及其实际发布时间；缺少任一条时，人民币篮子金额标记为不可用，而不是以离岸价格补齐。实时盘中可以单独显示参考汇率，但不得覆盖该笔赎回最终采用的 T 日官方收盘口径。

### 3.3 现金差额的现有样本检验

对本地缓存的 2026-05-11 至 2026-07-10 PCF，使用“后续首个 `PreTradingDay=T` 的 PCF”的 `CashComponent` 作为 T 日实际现金差额，共得到 42 个完整配对：

| 指标：实际现金差额 − T 日预估现金差额 | 数值（每 100 万份） |
| --- | ---: |
| 平均值 | +152.62 元 |
| 绝对误差中位数 | 353.32 元 / 4.31 bp |
| 绝对误差 P90 | 783.20 元 / 9.92 bp |
| 绝对误差 P95 | 886.68 元 / 10.92 bp |
| 最大绝对误差 | 1,678.87 元 / 18.41 bp |

这只是 42 个 PCF 的历史描述，不是可交易的 alpha，也不包括基金卖出相对收盘价的执行偏差。实盘保守阈值至少应把现金差额不确定性、股票卖出冲击、汇率、借券和两端手续费全部覆盖；在样本尚少时，采用 P95 至历史最大误差的压力测试，而非采用平均误差作收益加成。

## 4. 对冲方案：什么能对冲，什么不能锁定

### 4.1 不建议的理解

以下动作不能构成一笔已经锁定的赎回套利：

1. 中国盘中买入 159605 后，等到美国交易时段才做空 KWEB；
2. 港股收盘前买入港股成分股、美股收盘前买入美股成分股；
3. 同时保留 KWEB 空头并认为“个股回补后整体一定平”。

原因有三点：

- 你没有收到任何港股或 ADR 来交割空头；买入个股只是自行平掉此前的对冲空单。
- KWEB 与 159605 并非可按 1:1 兑换的同一合约。KWEB 当前持仓、权重、现金、费用、再平衡、交易时点和可借券成本均可能不同；KWEB 官方披露其自身持仓含港股及 ADR，并会在同一经济权益的美股/港股版本间选择。[KWEB 官方持仓页](https://kraneshares.com/etf/kweb/)
- 美国常规时段与中国 A 股时段不重叠。中国盘中买入 159605 后，到下一个美国开盘前的美股风险无法用常规时段 KWEB 当场锁定；即使使用盘前/盘后，流动性、点差及借券可得性仍需单独检验。

因此，“KWEB 空头 + 全量个股空头”会把重叠的中国互联网 beta 做成双重空头。若在随后把个股全部买回，残留的 KWEB 空头并不是自动可平的头寸，而是一笔独立的基差交易。

### 4.2 推荐的三层框架

**层 0：先建立结算样本，不下“无风险”结论。**

记录至少每笔实际赎回的 T 日、PCF 原件、申请确认、实际现金替代退款、实际现金差额、到账日、全部费用及基金是否有延迟/停牌。用 `V^settle_T` 校验 `\widehat V_t`，而不是只拿次日 NAV 或 IOPV 校验。

**层 1：PCF 精确成分股对冲（唯一可称为“主对冲”的方案）。**

若具备所有股票的可借券、交易权限与流动性，按当日 PCF 的 30 只证券、其原始股数及原始市场建立空头；不使用指数权重替代。将港股 23 只与 ADR 7 只分账，并保留各自的成交价格、时间和汇率。

- 港股腿：以 T 日港股收盘附近作为结算参考窗口；基金也可能在日内成交，故你的成交与基金实际成交仍有执行基差。
- 美国 ADR 腿：以 T 日美国常规收盘附近作为参考窗口；若以 15:59–16:00 的收盘窗口回补，只是在拟合“未卖出证券以收盘价结算”的兜底条款，**不是**对基金实际卖出时间的保证复制。
- 对冲平仓后，禁止再保留一笔没有量化用途的 KWEB 空头。

这套方案仍包含基金成交价、停牌/无交易证券处理、FX 估值汇率、股票借券召回、交易费用和到账周期风险；它是最小化 beta 的方案，不是确定收益保证。

**层 2：KWEB 代理对冲（只能作为 beta 对冲或流动性替代）。**

只有在直接成分股无法全量做空时，才考虑 KWEB。其名义数量不能由“一个篮子对应几股 KWEB”拍脑袋决定，应以历史数据回归得到：

\[
n_{KWEB}=\frac{\beta\times \widehat V^{US\ risk}}{P_{KWEB}\times USD/CNY}
\]

其中 `beta` 应分别在以下窗口估计并滚动复核：

- 中国买入时点至港股收盘；
- 港股收盘至美国开盘；
- 美国开盘至美国收盘；
- 整个 T 日最终退款相对 KWEB 的误差。

如果不具备足够的同窗口历史 Bid/Ask、KWEB 借券成本和实际退款样本，`beta` 应视为未知，而不是默认 1。KWEB 的官方资料显示其持仓跨香港、NASDAQ、NYSE，且同一经济权益可选择美股或港股版本，这正是代理误差的来源。[KWEB 说明书](https://kraneshares.com/resources/factsheet/kweb_factsheet.pdf)

**层 3：因子残差对冲（仅在层 1、层 2 已经实证后）。**

可用 `KWEB + 少数流动性好的 ADR/HK 龙头` 拟合 PCF 篮子，但必须用约束最小二乘/回测确定仓位，输出预测误差分布，并在交易前把 P95 误差计入折价门槛。不可先做 KWEB、再随意以个股“轧差”。

## 5. 建议的实际工作流

```text
盘前：下载并固化 T 日 PCF，检查申购=Y、赎回=Y、额度、三地共同开市
  ↓
中国盘中：以当日 PCF 的 30 只股票 + 双 FX 计算 Mid / 保守 Bid 两个篮子价值
  ↓
阈值：ETF Ask 与保守估值的折价，扣除 P95/压力误差、全部费用、借券和资金占用后仍为正
  ↓
买入 ETF 并提交赎回（记录申请确认）
  ↓
按已选择的“精确篮子”或“已校准 KWEB beta”做风险对冲；两者不得重复全额覆盖
  ↓
T+2：从后续 PCF 取 PreTradingDay=T 的 CashComponent；核对现金差额
  ↓
共同交易日结算后：取得实际现金替代退款，计算 R_T 与全部对冲 P&L，更新模型
```

### 必要的预交易检查

- 当日 PCF `Creation=Y`、`Redemption=Y`，并检查当日及净赎回上限；不要因为历史常常开放而假定一定可赎。
- 港股、美国及外汇市场开市；招募说明书把三地共同开市作为开放日条件。
- 全部 30 只成分股是否有可借券、可交易的合约、实时 Bid/Ask；停牌/缺价证券应将该笔标记为不可锁定。
- 赎回代理佣金（合同上限为 0.50%）、股票费率、港股印花/交易费、IB 借券费、FX 点差、保证金占用及到账时间。
- 券商退款字段的口径：是否包含实际现金差额、是否净额扣费、到账日期和交易日期分别是什么。

## 6. 后续网站接入规范

为避免误把 159605 当 159518，建议单独建立 `fund_mode = full_cash_substitution`：

| 字段 | 含义 |
| --- | --- |
| `pcf_trading_day` / `pcf_path` | 当日 PCF 版本与原始文件路径 |
| `cu_shares` | 1,000,000 |
| `hk_components` / `us_components` | PCF 分市场证券、原代码、数量、币种、实时 Bid/Ask、数据时间 |
| `estimated_cash_component` | T 日 `EstimateCashComponent` |
| `actual_cash_component` | 后续 PCF 中 `PreTradingDay=T` 的 `CashComponent`；缺失则为空 |
| `indicative_basket_mid` / `indicative_basket_low` | 实时估值，明确标为“指示性”，不可标为最终退款 |
| `cash_substitution_refund_actual` | 券商实收的现金替代退款 |
| `redemption_total_actual` | `refund + actual_cash_component - fees`，仅在字段口径已确认后计算 |
| `hedge_type` | `pcf_exact`、`kweb_proxy`、`factor_residual` 或 `none` |
| `hedge_error` | 最终退款与入场估值、以及对冲 P&L 的误差 |

页面上应把“PCF 成分篮子估值”和“预计赎回收入”分开显示：前者可实时更新，后者须标明 `EstimatedCashComponent`、执行价格和现金替代退款均未最终确定。不得把“预测整个篮子资产”命名为“预计退款”，除非已经取得本笔实际退款并完成口径校验。

## 7. 仍需补齐的数据，才能把方案升级为可交易模型

1. 至少 20–30 笔自己的 159605 实际赎回记录，包含每笔 `T / 退款 / 现金差额 / 所有费用 / 到账日`。
2. 对应日期全部 30 只股票的 1 分钟 Bid/Ask 或可重建的成交数据，以及 KWEB 的 regular / premarket / after-hours Bid/Ask。
3. 每笔执行的实际空头成交、回补成交、借券费、可借数量和被召回情况。
4. 外汇交易中心 T 日 `USD/CNY`、`HKD/CNY` 收盘记录及基金最终现金差额隐含汇率，用以复核基金所用的估值汇率规则。
5. 赎回代理券商对“现金替代退款”“现金差额”“其他对价”的账单口径书面确认。

在以上数据完成前，最合理的结论是：可以做 **PCF 实时指示性估值**，可以研究 KWEB 的代理 beta，但不能把当前折价直接解释为可通过赎回锁定的确定收益。
