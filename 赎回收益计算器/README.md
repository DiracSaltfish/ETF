# ETF 赎回收益计算器

## 用途

读取完整的 QMT1、QMT2 交割单和 IB 活动报表，自动计算 `159518` 每个赎回篮子的国内收益、XOP 对冲收益及人民币合计收益。

程序独立于 `V15-ETF`，只复用了其 PyQt5 界面风格和刷新结构。原有 LOF 赎回日期模块未参与本程序计算。

## 当前规则

1. QMT1、QMT2 被视为一个统一虚拟账户。
2. QMT 记录按 `交收日期 + 合同编号 + 来源账户 + 源行号` 全局排序；同日合同编号小的先处理。
3. `证券买入` 进入统一 FIFO 库存，成本取 `发生金额` 绝对值，因此包含交割单中的实际费用。
4. `证券卖出` 优先从 FIFO 库存剥离并单独计算场内平仓收益，不进入赎回篮子。
5. `ETF 基金赎回` 消耗剥离后的剩余 FIFO 库存。
6. `ETF 现金差额` 和 `ETF 申购退款` 按 `来源账户 + 合同编号` 归入对应篮子。
7. 每 `1,000,000` 份 `159518` 默认对应 `990` 股 XOP；IB 开仓 SELL 和平仓 BUY 默认分别按时间 FIFO 分配。
8. IB 佣金按实际成交记录计入；若只使用一笔成交的一部分，佣金和成交额按数量比例分摊。
9. IB 借券费按费用起息日仍处于未平仓状态的各篮子股数比例分摊。
10. 汇率由界面统一手动设置。人工指定某篮子 IB 成交后，覆盖记录保存在 `ib_mapping_overrides.json`。
11. 程序自动识别 `ETF 基金赎回`，按赎回日计算预计 T+3 现金差额日和 T+6 现金替代款到账日；周末及“休市日”中人工维护的日期均跳过。
12. 同日、不同 QMT 账户、方向相反、数量完全相同且合同号间隔不超过设置值的成交，自动识别为跨账户调仓；支持先卖后买和先买后卖。程序仍按合同号顺序执行，卖出消耗当时的旧 FIFO，买入形成新成本，调仓卖出不会进入普通场内平仓路径。

## 篮子配对图

“篮子配对图”用三栏同步展示匹配结果：左侧为 QMT1/QMT2 合并后的 `159518` 成交，中间为赎回篮子，右侧为 IB `XOP` 成交。

- 国内成交先按日期混合 QMT1/QMT2，再按合同编号、账户和源行号排序；`Seq` 是每天从 1 开始生成的合并日内顺序。
- 同一笔国内或 IB 成交若被拆给多个篮子，会按实际采用数量拆成多行，避免一笔成交只能显示一种篮子颜色。
- 篮子使用有限组浅色循环显示；左右成交与中间篮子同色，虚线连接各篮子在当前可见区域内的首尾成交。
- 灰色和同栏虚线括号表示国内自买自平，橙色表示 QMT1/QMT2 跨账户调仓，红色表示未进入篮子的 IB 自买自平，白色表示仍未分配。
- 三栏可独立上下滚动，透明绘图层会在滚动时动态重画连接线。顶部日期区间会分别按 QMT 成交日、赎回日和 IB 账单时间过滤。

## 结算状态

- `待现金差额`：未发现该合同对应的 `ETF 现金差额`。
- `待现金替代款`：已有现金差额，但尚未发现 `ETF 申购退款`。
- `IB未完全匹配`：990 股开仓或平仓不足。
- `国内库存不足`：统一 FIFO 库存不足以覆盖赎回份额。
- `已结算`：国内库存、IB 开平仓、现金差额和现金替代退款均完整。

未结算篮子仍显示暂算金额，但只有 `已结算` 篮子会进入顶部“已结算收益”合计。

## 到账日期与休市日

- 实际现金差额日取该篮子合同对应 `ETF 现金差额` 流水日期。
- 实际到账日取该篮子合同最后一笔 `ETF 申购退款` 流水日期。
- 预计现金差额日按赎回日后第 3 个工作日计算。
- 预计现金替代款到账日按赎回日后第 6 个工作日计算。
- 点击顶部“休市日”，可从日历添加或删除节假日；修改后所有篮子的预计日期会立即重算，实际流水日期和收益不受影响。
- 篮子汇总默认只显示预计 `T+3`、预计 `T+6到账`，并隐藏合同编号；点击顶部“显示详细”后展开预计/实际日期和合同编号。
- 在“篮子汇总”中双击任意篮子，可打开二级明细窗口，分别查看该篮子采用的国内 FIFO 买入、IB 做空开仓和 IB 回补平仓记录。

完整 QMT 交割单没有成交时间，因此跨账户调仓使用合同编号顺序作为时间代理。可在“数据源设置”调整“调仓合同号最大间隔”；自动规则采取精确数量匹配，未满足条件的成交保留在普通 FIFO 路径，避免过度识别。

## 本次实现经验

1. 可视化不能从汇总金额反推关系，计算引擎必须同时输出原始 QMT 成交、篮子 FIFO 分片、场内平仓分片、调仓 FIFO 分片和 IB 分片，UI 只消费这些可审计结果。
2. 一笔成交可能被多个篮子或特殊路径部分使用，因此颜色应绑定“采用数量分片”，不能直接绑定原始成交行。
3. QMT1/QMT2 调仓必须在普通场内卖出之前走独立路径，并同时支持先买后卖、先卖后买；目前使用同日、跨账户、反向、数量完全一致和合同号邻近五个保守条件。
4. Qt 可以完成滚动列表上的动态连线，不需要嵌入 Web 页面。透明覆盖层与表格是兄弟控件，坐标必须经全局坐标转换，不能把兄弟控件直接作为 `mapTo` 目标，否则会进入未定义行为。
5. IB 未分配成交需先扣除所有篮子分片，再按时间对剩余反向成交做 FIFO 自平配对；仍无法闭合的数量保持白色“未分配”，不能强行归篮子。
