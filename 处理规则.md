# ETF赎回套利处理规则

适用目录：`/Users/ellis/Desktop/ETF交割`

## 目标

后续在本目录处理 ETF 赎回套利时，统一按本文件口径执行，避免把场内日内平仓、赎回篮子、外盘对冲混在一起计算。

## 账户范围

- 国内账户：当前按 `2个QMT账户`
- 外盘账户：当前按 `1个IB账户`
- 后续支持多个国内账户和多个外盘账户并行流转

## 核心原则

### 1. QMT1/QMT2 按统一虚拟账户计算

- QMT1、QMT2 的开仓、场内卖出和赎回合并为一个统一虚拟账户
- 同一证券的两个 QMT 账户共享一套 FIFO 库存，但每笔记录仍保留来源账户用于审计
- 默认按 `成交日期 + 成交时间（如有）+ 合同编号/成交编号 + 来源账户` 从早到晚排序
- 完整交割单没有成交时间时，同一天合同编号小的先处理；合同编号相同再按来源账户稳定排序

### 2. 国内场内卖出要先剥离

- 如果某些国内买入仓位后续先被 `场内直接卖出平仓`，则这些仓位对应的买入记录必须先从买入库存中剥离
- 剥离规则按 QMT1/QMT2 合并后的统一账户、同一证券的 `FIFO` 执行
- 被场内卖出消耗掉的买入记录，**不得再进入 ETF 赎回篮子成本**

### 3. 场内卖出平仓收益单独计算

- 对于被场内卖出消耗的仓位，单独计算 `场内平仓收益`
- 计算方式：
  - `场内平仓收益 = 场内卖出金额净额 - 对应买入成本`
- 其中：
  - `场内卖出金额净额` 取该笔卖出对应的实际发生金额或净交收金额
  - `对应买入成本` 按 FIFO 从原始买入记录中分摊
- 这部分收益单独列表展示，不纳入赎回篮子收益

### 3.1 跨账户调仓单独识别

- 为凑足赎回份额而发生的 `QMT1卖出 + QMT2买入` 或反向操作，不作为普通场内平仓处理
- 完整交割单没有成交时间时，按以下保守条件自动识别：
  - 同一交收日、同一证券
  - 不同 QMT 来源账户
  - 买卖方向相反，支持先卖后买和先买后卖
  - 数量完全相同
  - 合同编号差不超过程序设置的最大间隔
- 调仓卖出仍按统一账户 FIFO 消耗旧库存并单列调仓已实现盈亏；调仓买入按实际发生金额形成新库存成本
- 调仓卖出不得重复进入普通场内卖出列表，调仓买入可继续进入后续赎回篮子 FIFO
- 未满足自动识别条件的成交保持普通路径，避免把无关交易过度识别为调仓

### 4. ETF赎回篮子必须唯一对应剩余买入记录

- 只有在剥离完场内卖出之后，剩余的国内买入库存，才可以用于匹配 `ETF基金赎回`
- 每轮赎回篮子成本，必须唯一对应到实际未被场内卖出消耗的买入记录
- 不允许把已经被场内平仓的买入记录再次计入赎回成本

### 5. 国内赎回成本仍按 FIFO

- 每轮赎回按 QMT1/QMT2 统一账户、同一证券的剩余库存做 FIFO 匹配
- 若某笔买入只使用部分数量，则按：
  - `分摊成本 = 原始发生金额绝对值 × 使用数量 / 原始买入数量`

### 6. 国内赎回到账金额

- 每轮赎回到账金额，仅统计和该轮赎回直接相关的资金流水正负净额
- 可包括：
  - `ETF赎回款`
  - `现金替代退补款`
  - `深市实时DS` 等赎回相关到账/回滚流水
- 不可混入：
  - `银行转证券`
  - `国债逆回购`
  - 其他无关证券交易资金

### 7. 现金差额单独列示

- `ETF现金差额扣款` 记为负贡献
- `ETF现金差额返款/补款` 记为正贡献
- 为避免重复计算，若现金差额已单独列示，则总赎回到账金额口径中不得再重复包含

### 8. 外盘对冲按账户单独处理

- IB 对冲记录按 `账户 + 标的` 单独处理
- 对冲交易默认按时间顺序做 FIFO 匹配
- 做空开仓为 `Sell`
- 平仓回补为 `Buy`
- 外盘盈亏：
  - `美元盈亏 = 卖出成交金额 - 买入成交金额 - 卖出佣金 - 买入佣金`

### 9. 国内赎回轮次与外盘对冲轮次需要显式映射

- 一轮国内赎回，不一定对应全部外盘交易
- 必须明确该轮赎回实际对应的 XOP 股数或对冲区间
- 如果用户指定例如 `100万份159518 对应 990股XOP`，则只按该指定股数和对应平仓区间计算
- 其余未纳入该轮的 XOP 交易，保留为未分配或归入下一轮

### 10. 多账户场景的处理要求

- 当前 QMT1/QMT2 按用户确认合并计算；若未来增加其他 QMT 或 IB 账户，必须保留以下字段：
  - `账户类型`
  - `账户名称/账号`
  - `证券代码/标的`
  - `成交日期`
  - `成交时间`
  - `方向`
  - `数量`
  - `发生金额/净额`
- QMT1/QMT2 的 FIFO、赎回匹配和场内平仓匹配跨账户统一计算；IB 仍按实际账户分别处理

## 输出要求

后续每轮至少拆成三部分输出：

### A. 国内场内平仓明细

- 账户
- 证券代码
- 卖出日期时间
- 卖出数量
- 对应买入明细
- 对应买入成本
- 场内卖出净额
- 场内平仓收益

### B. ETF赎回篮子明细

- 轮次
- 账户
- 赎回日期
- 赎回份额
- 被采用的国内买入明细
- ETF买入成本
- 赎回到账金额
- 现金差额
- 国内盈亏

### C. 外盘对冲明细

- 轮次
- IB账户
- 标的
- 匹配股数
- 做空开仓区间
- 平仓区间
- 美元盈亏
- 汇率
- 人民币盈亏

## 总盈亏口径

每轮总盈亏：

`总盈亏 = 国内赎回到账金额 - 国内ETF买入成本 + 现金差额贡献 + 外盘人民币盈亏 - 其他费用`

注意：

- `场内平仓收益` 单独统计，不纳入赎回篮子收益
- 如果用户要求看总策略收益，可额外输出：
  - `总策略收益 = 场内平仓收益 + 赎回篮子收益`

## 未完全匹配处理

以下任一情况都必须标记为 `未完全匹配`：

- 赎回份额无法由剩余国内买入库存完全覆盖
- 场内卖出数量无法从历史买入中完全匹配
- 外盘对冲股数无法和该轮指定股数完全匹配
- 多账户之间缺少明确映射关系

出现未完全匹配时：

- 不强行合并
- 单独列出缺口数量、涉及账户、涉及时间段
- 保留已匹配部分和未匹配部分的分界

## 当前已确认的特殊口径

- `100万份 159518` 的赎回轮次，外盘对冲口径按用户指定：`对应 990股 XOP`
- 如果未来用户再次指定某轮赎回对应的外盘股数或指定平仓区间，以用户指定为准
