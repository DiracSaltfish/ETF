# 159518 盘中总篮子资产估值方案

更新日期：2026-07-10。本文定义 159518 的可复用估值口径；它服务于“买入 159518、同时做空 XOP、赎回后在美东尾盘回补 XOP”的折价套利监控与结算复核。

## 结论

应使用**总篮子资产**，而不是只用 `998 × XOP × 汇率` 的证券退款代理。

赎回最终收到的人民币资产是：

```text
篮子资产 = ETF 申购退款 + ETF 现金差额
```

因此，盘中可比较的单位份额估值也必须把 PCF 的预估现金差额纳入。当前可用的总资产代理是：

```text
证券资产 = 996 × XOP 价格 × USD/CNY
总篮子资产 = 证券资产 + 当日 PCF EstimateCashComponent
每份总资产预估净值 = 总篮子资产 ÷ 1,000,000
```

其中 996 是 1,000,000 份 159518 对应的**证券资产 XOP 等价股数**，不是当前历史账务或条件单的 990 股固定数量。

## 两套必须分开的估值口径

### A. 中国盘中实时估值：用于判断是否存在可执行折价

设：

- `q = 申赎份额 / 1,000,000`；实时盘口默认 `q = 1`
- `S = 996`（每个最小申赎单位的 XOP 证券资产等价股数）
- `B` / `A` = XOP 实时 Bid / Ask，美元
- `F(t)` = 此时中国外汇交易中心已发布的最新 USD/CNY 小时参考价
- `C` = 当日 159518 PCF 的 `EstimateCashComponent`，人民币/最小申赎单位

则：

```text
Bid 总篮子资产 = q × (S × B × F(t) + C)
Ask 总篮子资产 = q × (S × A × F(t) + C)
Bid / Ask 单位净值 = 对应总篮子资产 ÷ (q × 1,000,000)
```

对“买入 159518、同步做空 XOP”的方向，页面中最应关注的保守盘口是：

```text
买入方向溢折价率 = 159518 卖一 ÷ Bid 单位净值 - 1
```

该值为负才是折价；实际是否下单还要扣除佣金、借券费、冲击成本、申赎成本及模型安全垫。

实时页面不把缺失的 PCF 现金差额视为零：当日 PCF 未缓存或缺少 `EstimateCashComponent` 时，停止输出总资产溢折价，以免出现方向性误判。

### B. T 日结算预测：用于赎回后复核，不用于中国盘中假装已知

在当前 5 个独立实际结算日的样本中，最优的总资产匹配为：

```text
预测总篮子资产
= q × (996 × XOP 美东 15:59 一分钟收盘价 × T 日 CFETS 16:00 USD/CNY
       + PCF EstimateCashComponent)
```

这不是盘中已知的价格。中国盘中只能看到当前 XOP Bid/Ask 和当时已发布的 CFETS 价；美东 15:59 价格仅能在之后用于结算预测/回测。因此实时页面使用 A，篮子汇总的“预估篮子资产”使用 B。

`16:00` 是当前小样本中误差最低的**CFETS 时点代理**，并不证明基金公司合同中的“外汇交易中心收盘价”就是 16:00。若能取得基金公司实际采用的正式收盘汇率或更长的到账样本，应以该正式规则覆盖 `PREDICTED_BASKET_FX_QUOTE_TIME` 后重新回测；不能把 5 个样本的拟合优胜视为结算规则本身。

## 回测依据与限制

样本为 2026-06-22、06-23、06-25、06-29、06-30 五个独立确认的结算日；6 月 23 日两笔相同退款按一个独立日处理。

- PCF 成分证券市值（不含现金差额）相对 XOP 的等价股数中位数：**996 股**。
- 固定 996 股、XOP 美东 15:59 一分钟收盘价、T 日 CFETS 16:00、加 PCF `EstimateCashComponent` 的总资产模型：样本内 RMSE **697.35 元**，MAE **572.00 元**。
- 若强制改用本地 `CLOSE` 字段而非 16:00，最佳相同价格点模型 RMSE 为 **920.33 元**。

这些数字说明“总资产 + 996 股”比只看退款腿更贴近本样本的结算对象，但只有 5 个独立日，不能视作无风险或精确到秒的成交证明。PCF 现金差额本身也会有预测误差；它不应被错误归因于 XOP 15:59 的执行误差。

## 需要的数据与刷新规则

| 输入 | 实时用途 | 结算预测用途 | 程序来源 |
|---|---|---|---|
| XOP Bid/Ask | 证券资产实时 Bid/Ask | 不使用 | IB TWS 流式行情 |
| XOP 美东 15:59 一分钟收盘价 | 不使用 | 证券资产结算代理 | TWS 历史 `TRADES` 1 分钟线 |
| 159518 买一/卖一、五档 | 溢折价与可执行门槛 | 不使用 | 新浪 `sz159518`，3 秒轮询 |
| `EstimateCashComponent` | 总篮子资产现金腿 | 同一现金腿 | 当日深交所 159518 PCF |
| CFETS USD/CNY 最新小时价 | 中国盘中实时换算 | 不使用 | 中国外汇交易中心 |
| CFETS USD/CNY 16:00 | 不使用 | T 日结算预测固定汇率 | 中国外汇交易中心 |
| 实际退款、实际现金差额 | 不使用 | 复核预估误差 | QMT 交割单/手工确认 |

实时 CFETS 客户端每分钟刷新一次。当日 API 已出现新的小时字段（当前数据会出现 10:00、11:00、14:00、15:00、16:00、17:00、18:00 等）后，自动切换至最新可用时点；当天尚未发布时使用最近一个有数据的交易日，并在页面和 JSON 中保留实际的 `trading_day` 和 `quote_time`。结算预测则固定取 T 日 `16:00`，不随之后的 17:00/18:00 更新改写。

PCF 页面的“当日实际现金差额”用于单独验证现金腿：当查看 T 日 PCF 时，程序只扫描本地已缓存的后续 PCF，寻找 `PreTradingDay = T` 的首个文件，并显示其中的 `CashComponent` 及来源 PCF 日期。例如，2026-06-29 的实际现金差额会从 2026-07-01 PCF 读取。没有后续缓存时该框留空，且不会为此发起网络请求。

## 页面与文件实现

### 实时溢价率

页面现在显示四层可审计计算：

1. 996 股 XOP 证券资产等价；
2. XOP 证券资产的人民币 Bid / Ask 值；
3. 当日 PCF 预估现金差额；
4. 总篮子资产、单位总资产净值和盘口溢折价率。

共享的 `realtime.json` 已升级至 schema version 2，新增：

```text
valuation.pcf.estimate_cash_component_cny
valuation.pcf.trading_day
valuation.stock_component.bid_cny / ask_cny
valuation.basket.bid_cny / ask_cny
valuation.nav.bid / ask
order_book.*.premium_rate_vs_basket_bid_nav / ask_nav
```

`valuation.nav` 及订单簿中所有溢折价率均已基于总篮子资产。为兼容旧读取程序，原有 `premium_rate_vs_xop_*_nav` 字段仍输出同值别名；schema 2 中它们不再表示“仅 XOP 资产”。

### 篮子汇总

“篮子汇总”改为依次显示：

```text
现金差额 | 退款 | 篮子资产 | 预估篮子资产 | 预估偏差
```

```text
篮子资产 = 实际退款 + 实际现金差额
预估篮子资产 = 996 × XOP 15:59 一分钟收盘价 × T 日 CFETS 16:00
             + 当日 PCF EstimateCashComponent
预估偏差 = 篮子资产 ÷ 预估篮子资产 - 1
```

只有退款和现金差额都已实际入账（或退款被人工确认）时才显示预估偏差；尚未结算的篮子显示 `--`，不把未到账金额误认为零。默认摘要中的赎回日、T+3、T+6 只显示 `MM-DD`；点“显示详细”后显示完整年份和实际到账日期。

旧的仅退款缓存会被识别为旧口径，不会误显示为预估篮子资产。点击“预测重算”会重新读取 XOP 15:59、CFETS 16:00 与 PCF 后写入新口径。

## 对冲数量的操作边界

996 是当前估值与结算预测的最佳**资产等价系数**。程序没有自动把历史账务、IB 分配或晚间条件单从 990 改为 996：这些模块的既有对账口径仍是 990，直接改动会改变历史盈亏归因和真实下单规模。

若要把晚间回补默认数量改为 996，应先用更多实际赎回日验证：

1. 实际退款对应的 XOP 股数是否稳定围绕 996；
2. 现金差额预测误差是否与股票腿误差分离；
3. 996 对冲下的实际残余敞口、借券费和成交冲击是否优于 990。

在此之前，实时总资产估值采用 996，但交易规模仍由现有风险控制和人工确认决定。
